**システム要件定義書**

1日タイムライン記録アプリ

バージョン 1.0　\|　作成日：2026年3月20日

**1. プロジェクト概要**

  ---------------------- -----------------------------------------------------------------------------
  プロジェクト名         1日タイムライン記録アプリ

  目的                   1日の行動を時刻付きで記録し、終了後に一覧で振り返ることができる個人用ツール

  対象ユーザー           個人（作成者本人）

  実行環境               ブラウザでHTMLファイルを直接開いて使用（サーバー不要）

  技術スタック           HTML / CSS / JavaScript（単一ファイル）

  バージョン管理         対象外（個人利用のため）
  ---------------------- -----------------------------------------------------------------------------

**2. 機能要件**

**2.1 画面構成・画面遷移**

本アプリは単一ページで構成し、状態によって表示内容を切り替える。

  ---------- -------------- ------------------------------------------------------------------------------------------
  **状態**   **画面名**     説明

  S-01       スタート画面   アプリ起動時に表示。「スタート」ボタンのみを表示する

  S-02       記録中画面     スタート後に表示。開始時刻・入力フォーム・登録ボタン・記録中リスト・終了ボタンを表示する

  S-03       一覧表示画面   終了ボタン押下後に表示。全記録の一覧・コピーボタン・リセットボタンを表示する
  ---------- -------------- ------------------------------------------------------------------------------------------

**2.2 機能一覧**

  -------- ------------------ ------------------------------------------------------------------------------------------------------------------------------------------------------------
  **ID**   **機能名**         詳細説明

  F-01     計測開始           「スタート」ボタンを押した時点の時刻を記録開始時刻として保持し、記録中画面へ遷移する

  F-02     行動入力           テキスト入力フォームに行動内容（例：「メール」）を入力できる

  F-03     行動登録           「登録」ボタン押下、またはEnterキーで行動を登録する。登録時の時刻を自動取得し「〜HH:MM 行動名」形式で保存する

  F-04     記録中プレビュー   登録済みの行動を記録中画面の下部に随時リスト表示する（最新が下に追加される）

  F-05     記録の削除         記録中画面において、各行動エントリの横に削除ボタンを設置し、誤入力を取り消せる

  F-06     終了・一覧表示     「終了」ボタン押下で記録を確定し、全行動の一覧を時系列で表示する

  F-07     テキストコピー     一覧表示画面に「コピー」ボタンを設置し、全記録をテキスト形式でクリップボードにコピーできる。コピー成功時はボタンラベルを一時的に「コピー完了！」に変更する

  F-08     リセット           「リセット」ボタン押下でアプリを初期状態（スタート画面）に戻す。押下前に確認ダイアログを表示する
  -------- ------------------ ------------------------------------------------------------------------------------------------------------------------------------------------------------

**2.3 表示フォーマット**

行動の表示形式は以下の通り統一する。

  ------------------------ ------------------------------------------------
  項目                     仕様

  時刻フォーマット         HH:MM（24時間表記）

  行動表示形式             〜HH:MM　行動名

  一覧の並び順             登録順（時系列の昇順）

  コピー時のテキスト形式   〜HH:MM 行動名 を改行区切りで連結した平文
  ------------------------ ------------------------------------------------

**3. 非機能要件**

  ---------------------- ----------------------------------------------------------------
  項目                   仕様

  データ永続化           なし。ページをリロード・閉じると記録は消える（セッション限り）

  外部依存               なし。外部ライブラリ・API・サーバー不使用

  対応ブラウザ           最新版のChrome / Safari / Edge（PC）

  レスポンシブ対応       対象外（デスクトップ利用を前提とする）

  パフォーマンス         記録件数100件程度までの動作を保証。それ以上は対象外
  ---------------------- ----------------------------------------------------------------

**4. UI・デザイン要件**

  ---------------------- ------------------------------------------------------------------------------
  項目                   方針

  デザインテイスト       シンプル・ミニマル。余計な装飾を排除した機能優先のUI

  カラーパレット         背景：白（#FFFFFF）、テキスト：黒〜グレー系、アクセント：単色（例：#333333）

  フォント               システムフォント（sans-serif）を使用

  ボタン配置             操作の流れに沿って上から下へ自然に並べる

  フィードバック         コピー完了時のボタンラベル変更、削除時の即時更新
  ---------------------- ------------------------------------------------------------------------------

**5. 操作フロー（ユースケース）**

以下が標準的な1日の使用シナリオ。

- アプリ（HTMLファイル）をブラウザで開く

- 「スタート」ボタンを押す → 記録中画面へ遷移

- 行動内容（例：「会議」）を入力して「登録」ボタン押下 or Enterキー

- 「〜09:30　会議」として記録される

- 以降、同様に行動が切り替わるたびに入力・登録を繰り返す

- 誤入力があった場合は各行の削除ボタンで取り消す

- 1日の終わりに「終了」ボタンを押す → 一覧表示画面へ遷移

- 一覧を確認し、「コピー」ボタンでテキストをクリップボードへ取得

- 必要であれば「リセット」で初期画面へ戻す

**6. 制約事項・対象外機能**

- データのサーバー保存・クラウド同期は対象外

- ユーザー認証・ログイン機能は対象外

- スマートフォン・タブレットのレスポンシブ対応は対象外

- 過去データの参照・比較機能は対象外

- カテゴリ分類・タグ付けは対象外

- CSV・PDF等のファイルエクスポートは対象外（テキストコピーのみ）
